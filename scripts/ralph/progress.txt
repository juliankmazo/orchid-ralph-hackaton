## Codebase Patterns
- Server project lives under `server/` with TypeScript, Express, and pg
- Use `npx tsc --noEmit` from `server/` directory to typecheck
- Database connection uses `pg.Pool` with `DATABASE_URL` env var
- Express app exported from `src/index.ts`, db pool from `src/db.ts`
- Migrations live in `server/migrations/` as plain SQL, run via `src/migrate.ts` on startup
- Use `CREATE TABLE IF NOT EXISTS` for idempotent migrations
- Server startup is async via `start()` function in index.ts
- Auth middleware `requireApiKey` in index.ts — reuse for all protected endpoints
- Upsert pattern: `INSERT...ON CONFLICT (id) DO UPDATE SET...RETURNING *`
