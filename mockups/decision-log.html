✅ Chose PostgreSQL over MongoDB

Use PostgreSQL as the primary database for storing conversation transcripts and session metadata.

Alternatives considered
MongoDB SQLite
Why

PostgreSQL offers excellent JSONB support and handles relational joins natively — essential for linking sessions, commits, and users without application-level joins.

✅ Dumb write, smart read — raw transcript storage

Store the full JSONL transcript as-is with no parsing or processing at write time.

Alternatives considered
Parse at ingest Incremental diffs Summarize on upload
Why

Keeps the write path simple and crash-safe. All intelligence (linking to commits, answering questions) happens at read time, so new features can be added without migrating existing data.

✅ Polling every 5s instead of WebSockets

Sync transcripts to the server via HTTP polling every 5 seconds rather than a persistent WebSocket connection.

Alternatives considered
WebSockets SSE Post-session upload
Why

Polling is stateless and crash-safe with no reconnection logic needed. Indistinguishable from real-time for this use case. WebSockets can be added later if sub-second latency is required.

✅ Plain TypeScript for CLI — no Commander or Oclif

Use plain TypeScript with manual argument parsing instead of a CLI framework.

Alternatives considered
Commander Oclif Yargs
Why

For a 4-command CLI, frameworks add unnecessary dependencies and learning curve. A switch on argv[2] is all that's needed. Migrate later if the command surface grows.
